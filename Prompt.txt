1. ระบบปฏิทินโรงเรียนแบบยืดหยุ่น
   ==================================================

ชื่อโมดูล:
School Calendar & Flexible Academic Schedule

ชื่อเมนู:
ปฏิทินโรงเรียน

ตำแหน่งเมนู:
ตั้งค่า
├── ปีการศึกษา
├── ภาคเรียน
├── ปฏิทินโรงเรียน
└── วันเรียน / วันหยุด / กิจกรรมพิเศษ

แนวคิด:
ระบบต้องมีปฏิทินโรงเรียนเพื่อช่วยจัดการวันเรียน วันหยุด วันกิจกรรม วันสอบ วันเปิด/ปิดภาคเรียน และวันเรียนชดเชย แต่ต้องไม่บังคับว่าทุกโรงเรียนต้องเช็คชื่อตามปฏิทินแบบตายตัว เพราะแต่ละโรงเรียนมีบริบทต่างกัน

หลักสำคัญ:

* โรงเรียนเป็นผู้ตั้งค่าเอง
* ระบบไม่ควรบังคับเช็คชื่อแบบแข็ง ๆ
* ปฏิทินมีไว้ช่วยแนะนำ/ช่วยกรอง/ช่วยรายงาน
* โรงเรียนสามารถกำหนดเสาร์-อาทิตย์ให้เป็นวันเรียนได้
* โรงเรียนสามารถกำหนดวันธรรมดาให้เป็นวันหยุดได้
* โรงเรียนสามารถกำหนดวันกิจกรรมพิเศษได้
* โรงเรียนสามารถเช็คชื่อในวันใดก็ได้ หากมีสิทธิ์และตั้งค่าอนุญาต
* ต้องมี override ได้เสมอ

ประเภทวันในปฏิทิน:

* school_day = วันเรียนปกติ
* holiday = วันหยุด
* public_holiday = วันหยุดราชการ
* makeup_day = วันเรียนชดเชย
* activity_day = วันกิจกรรมพิเศษ
* exam_day = วันสอบ
* semester_start = วันเปิดภาคเรียน
* semester_end = วันปิดภาคเรียน
* custom = กำหนดเอง

ตัวอย่าง:

* วันเสาร์ที่ 15 มิถุนายน 2569 = วันเรียนชดเชย
* วันจันทร์ที่ 22 กรกฎาคม 2569 = วันหยุดโรงเรียน
* วันศุกร์ที่ 30 สิงหาคม 2569 = วันกิจกรรมวันแม่
* วันที่ 1-5 กันยายน 2569 = วันสอบกลางภาค

ฟังก์ชัน:

* เพิ่มวันเรียน
* เพิ่มวันหยุด
* เพิ่มวันกิจกรรม
* เพิ่มวันสอบ
* เพิ่มวันเรียนชดเชย
* เพิ่มช่วงเปิด/ปิดภาคเรียน
* ตั้งค่าทั้งปีการศึกษา
* import ปฏิทินจาก Excel
* คัดลอกปฏิทินจากปีที่แล้ว
* แสดงปฏิทินรายเดือน
* ใช้สีแยกประเภทวัน
* เพิ่มหมายเหตุรายวัน
* เปิด/ปิดผลต่อระบบเช็คชื่อได้

การเชื่อมกับระบบเช็คชื่อ:
ระบบต้องมีตัวเลือก:

1. ใช้ปฏิทินเพื่อแนะนำเท่านั้น
2. ใช้ปฏิทินเพื่อแจ้งเตือนว่าวันนี้ควร/ไม่ควรเช็คชื่อ
3. ใช้ปฏิทินเพื่อกรองรายงาน
4. ไม่บังคับเช็คชื่อจากปฏิทิน

ตัวอย่างข้อความ:

* “วันนี้ถูกตั้งเป็นวันหยุด แต่ครูยังสามารถเช็คชื่อได้หากเป็นวันเรียนชดเชยหรือกิจกรรมพิเศษ”
* “วันนี้เป็นวันเรียนชดเชย ระบบอนุญาตให้เช็คชื่อได้”
* “วันนี้เป็นวันกิจกรรมพิเศษ ครูสามารถบันทึกสถานะกิจกรรมแทนการมาเรียนปกติได้”

2. ระบบป้องกันข้อมูลซ้ำตอน Import
=================================

ชื่อโมดูล:
Import Preview & Duplicate Protection

ชื่อเมนู:
ตรวจสอบก่อนนำเข้า

เป้าหมาย:

* ป้องกันข้อมูลพังจาก Excel
* ตรวจข้อมูลซ้ำก่อนนำเข้าจริง
* ให้ครูเห็น preview ก่อน confirm
* แสดง error/warning รายแถว
* รองรับปี พ.ศ.
* รองรับการอัปเดตข้อมูลเดิมแบบ upsert

Flow Import:

1. ครูอัปโหลดไฟล์ Excel/CSV
2. ระบบอ่านข้อมูลแบบ preview
3. ตรวจหัวคอลัมน์
4. ตรวจชนิดข้อมูล
5. ตรวจข้อมูลจำเป็น
6. ตรวจข้อมูลซ้ำในไฟล์
7. ตรวจข้อมูลซ้ำกับฐานข้อมูลเดิม
8. แสดงรายการ error/warning
9. ให้ครูเลือกวิธีจัดการ:

   * เพิ่มเฉพาะข้อมูลใหม่
   * อัปเดตข้อมูลเดิม
   * ข้ามแถวที่ผิด
   * ยกเลิก
10. ครูกดยืนยัน
11. ระบบนำเข้าจริง
12. สร้าง import_job และ import_rows log

ข้อมูลที่ต้องตรวจ:

* เลขประจำตัวนักเรียนซ้ำ
* เลขบัตรประชาชนซ้ำ
* ชื่อ-นามสกุลซ้ำ
* วันเกิดผิดรูปแบบ
* ปี พ.ศ. แปลงผิด
* ห้องเรียนไม่ตรงกับระบบ
* เลขที่ซ้ำในห้อง
* ข้อมูลผู้ปกครองว่าง
* เบอร์โทรผิดรูปแบบ
* ยอดเงินออมไม่ใช่ตัวเลข
* คะแนนเกินคะแนนเต็ม
* รหัสวิชาไม่ตรง
* วันที่เป็นอนาคต/ผิดรูปแบบ

ระดับผลตรวจ:

* error = ต้องแก้ก่อนนำเข้า
* warning = นำเข้าได้แต่ควรตรวจ
* info = ข้อมูลแจ้งให้ทราบ

ตัวอย่าง:

* ERROR: เลขบัตรประชาชนไม่ครบ 13 หลัก
* ERROR: วันเกิด 31/02/2558 ไม่มีอยู่จริง
* WARNING: พบชื่อ-นามสกุลซ้ำกับนักเรียนเดิม
* WARNING: ไม่มีเบอร์โทรผู้ปกครอง
* INFO: นักเรียนรหัส 001 มีอยู่แล้ว ระบบจะอัปเดตข้อมูล

3. ระบบ Undo / ถังขยะ / กู้คืนข้อมูล
====================================

ชื่อโมดูล:
Trash, Undo & Restore Center

ชื่อเมนู:
ถังขยะ / กู้คืนข้อมูล

ตำแหน่งเมนู:
เครื่องมือระบบ
├── ถังขยะ
├── กู้คืนข้อมูล
└── ประวัติการลบ

เป้าหมาย:

* ป้องกันการลบข้อมูลผิดพลาด
* ให้ครูกู้คืนข้อมูลได้ตามสิทธิ์
* ลดความกลัวในการใช้งานระบบ
* มี audit log ทุกครั้ง
* ลบถาวรได้เฉพาะ admin/owner/superadmin

หลักการ:

* ใช้ soft delete ด้วย deleted_at
* ข้อมูลที่ลบจะเข้า Trash
* กู้คืนได้ภายในระยะเวลาที่กำหนด เช่น 30 วัน
* ลบถาวรได้เฉพาะผู้มีสิทธิ์
* ไฟล์ใน R2 ยังไม่ลบทันที ให้ mark deleted ก่อน
* ถ้าลบถาวรแล้วจึง cleanup ไฟล์จริง

รายการที่ควรกู้คืนได้:

* นักเรียน
* ห้องเรียน
* คะแนน
* งาน/ใบงาน
* งานค้าง
* เงินออม transaction
* บันทึกเวลาเรียน
* พฤติกรรม
* บันทึกครูประจำชั้น
* นัดหมายผู้ปกครอง
* แบบฟอร์ม
* ตารางเวร
* ผลการทำเวร
* ไฟล์แนบ
* รายงาน

ฟังก์ชัน:

* ดูรายการที่ถูกลบ
* filter ตามประเภทข้อมูล
* filter ตามผู้ลบ
* filter ตามวันที่ลบ
* ดูรายละเอียดก่อนกู้คืน
* กู้คืนรายการเดียว
* กู้คืนหลายรายการ
* ลบถาวร
* ลบถาวรอัตโนมัติหลังครบ retention
* บันทึกเหตุผลเมื่อลบถาวร

4. ระบบแจ้งเตือนข้อมูลผิดปกติขั้นสูง
====================================

ชื่อโมดูล:
Advanced Data Health Check

ชื่อเมนู:
ตรวจสุขภาพข้อมูล

เป้าหมาย:

* ตรวจข้อมูลผิดปกติอัตโนมัติ
* ช่วยครูแก้ก่อนสร้างรายงาน
* ป้องกันข้อมูลพัง
* เตือนก่อนเปิด Public Portal
* เตือนก่อนปิดงวด/ปิดปี

รายการตรวจ:
ข้อมูลนักเรียน:

* นักเรียนไม่มีห้อง
* นักเรียนไม่มีเลขที่
* นักเรียนไม่มีวันเกิด
* นักเรียนไม่มีเลขบัตรประชาชน
* เลขบัตรประชาชนไม่ครบ 13 หลัก
* เลขบัตรประชาชนซ้ำ
* วันเกิดผิดรูปแบบ
* นักเรียนซ้ำจากชื่อ-นามสกุล

Public Portal:

* เปิด Public Portal แต่ข้อมูลยืนยันไม่ครบ
* นักเรียนไม่มี birth_date
* นักเรียนไม่มี citizen_id_hash
* ตั้งค่า school_code ซ้ำ
* มีการกรอกผิดหลายครั้งผิดปกติ

เงินออม:

* ยอดเงินออมติดลบ
* current_balance ไม่ตรงกับ transaction
* opening_balance ซ้ำ
* transaction วันที่ผิดปีการศึกษา
* ฝาก/ถอนเป็นค่าติดลบผิดประเภท

คะแนน:

* คะแนนเกินคะแนนเต็ม
* คะแนนติดลบ
* งานไม่มีคะแนนเต็ม
* งานไม่มีตัวชี้วัด
* คะแนนซ้ำ
* ตัวชี้วัดไม่ผูกงาน

เวลาเรียน:

* เช็คชื่อซ้ำ
* นักเรียนไม่มี record ในวันที่ควรมี
* วันที่เช็คชื่ออยู่นอกปีการศึกษา
* สถานะไม่ถูกต้อง

เวร:

* เวรวันนี้มีนักเรียนขาดแต่ยังไม่มีคนแทน
* ตารางเวรซ้ำ
* นักเรียนได้เวรหนักซ้ำเกินเงื่อนไข
* ยังไม่ได้บันทึกผลการทำเวร
* duty_rule ไม่มีวันทำเวร
* duty_rule จำนวนคนมากกว่านักเรียนที่มี

ไฟล์:

* file_assets มี metadata แต่ R2 object หาย
* ไฟล์หมดอายุแต่ยังไม่ cleanup
* สลิปชำระเงินเกิน retention
* backup หมดอายุ

ระบบ:

* subscription หมดอายุแต่ยัง active
* refunded แล้วแต่ยังเปิด VIP
* referral credit ผิด
* notification ส่งไม่สำเร็จจำนวนมาก

7. ระบบข้อความตัวอย่าง / เทมเพลตข้อความ
=======================================

ชื่อโมดูล:
Parent Message Templates

ชื่อเมนู:
เทมเพลตข้อความ

ตำแหน่งเมนู:
ตั้งค่า
├── เทมเพลตข้อความ

หรือ:
ดูแลนักเรียน
├── ข้อความถึงผู้ปกครอง
├── เทมเพลตข้อความ

เป้าหมาย:

* ให้ครูส่งข้อความถึงผู้ปกครองได้เร็ว
* ลดเวลาพิมพ์ข้อความซ้ำ
* ใช้กับ LINE/Telegram/In-App/คัดลอกข้อความ
* รองรับตัวแปรอัตโนมัติ
* ครูแก้ไขก่อนส่งได้

ประเภทเทมเพลต:

* แจ้งงานค้าง
* แจ้งขาดเรียน
* แจ้งมาสาย
* แจ้งเงินออม
* แจ้งนัดหมาย
* แจ้งประชุมผู้ปกครอง
* แจ้งตอบแบบฟอร์ม
* แจ้งพฤติกรรมดี
* แจ้งพฤติกรรมที่ต้องติดตาม
* แจ้งเวร
* แจ้งกิจกรรมโรงเรียน
* แจ้งรายงานประจำเดือน

ตัวแปรที่รองรับ:

* {{student_name}}
* {{classroom_name}}
* {{teacher_name}}
* {{school_name}}
* {{date_th}}
* {{missing_work_count}}
* {{attendance_status}}
* {{savings_balance}}
* {{appointment_date}}
* {{form_due_date}}
* {{duty_name}}
* {{portal_link}}

ตัวอย่างเทมเพลต:

แจ้งงานค้าง:
เรียนผู้ปกครองของ {{student_name}} ขณะนี้มีงานค้างจำนวน {{missing_work_count}} งาน กรุณาตรวจสอบรายละเอียดในระบบ ClassCare 360 หรือติดต่อครูประจำชั้น ขอบคุณค่ะ/ครับ

แจ้งขาดเรียน:
เรียนผู้ปกครองของ {{student_name}} วันนี้ {{date_th}} ระบบบันทึกสถานะการมาเรียนเป็น “{{attendance_status}}” หากมีข้อผิดพลาด กรุณาติดต่อครูประจำชั้น

แจ้งเงินออม:
เรียนผู้ปกครองของ {{student_name}} ยอดเงินออมคงเหลือล่าสุดคือ {{savings_balance}} บาท สามารถตรวจสอบข้อมูลสรุปได้ผ่านระบบ ClassCare 360

แจ้งนัดหมาย:
เรียนผู้ปกครองของ {{student_name}} โรงเรียนขอนัดหมายเพื่อพูดคุยในวันที่ {{appointment_date}} กรุณายืนยันหรือติดต่อครูประจำชั้นหากต้องการเลื่อนนัด

แจ้งตอบแบบฟอร์ม:
เรียนผู้ปกครองของ {{student_name}} มีแบบฟอร์มที่ต้องตอบภายในวันที่ {{form_due_date}} กรุณาตรวจสอบและตอบกลับผ่านระบบ ClassCare 360
